Machine Learning Survival Models

Random Survival Forest, Gradient Boosted Survival, SHAP interpretability

Run setup Train one RSF or GBS model here. Compare All below is a separate screening run.
Evaluation These settings apply to Compare All screening only. Train Model uses its own holdout/apparent path.
Compare All is a screening comparison across Cox PH, RSF, and GBS on one shared evaluation path. Train Model fits one RSF or GBS model and reports its own holdout/apparent evaluation mode in the result banner. Fast mode skips SHAP so RSF returns much faster on larger cohorts.
Partial dependence and counterfactual analysis are available for tree-model runs only (RSF or GBS). They reuse the latest matching trained tree model when available; otherwise they refit the same requested tree settings.

Deep Learning Survival Models

DeepSurv, DeepHit, Neural MTLR, Survival Transformer, Survival VAE

Run setup The minimum needed for one deep-learning run.
Architecture Optimization and network-shape controls for the selected model.
Validation and runtime Used mainly when repeated CV or longer training is needed.
Model-specific advanced controls Shown only when the selected architecture actually uses them.
Evaluation Mode applies to both Train Model and Compare All. Holdout is the fastest default and returns one fitted model with plots. Repeated CV is slower, returns aggregate performance, and skips single-fit plots such as one loss curve or one feature-importance chart.
Allowed ranges: Epochs 10-1000, Learning rate 0-0.1, Dropout 0-0.5, Batch size 8-512, CV folds 2-10, CV repeats 1-20, Patience 1-100, Min delta 0-0.1, and Parallel jobs 1-16. Hidden layers use the full comma-separated stack such as 64,64.
